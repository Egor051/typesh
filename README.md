# Discord-бот виджета Breaking Proxy

Лёгкий Python-бот, который:
- загружает HTML со страницы `breaking.proxy.sqstat.ru`,
- парсит **только две карточки серверов**: `RAAS/AAS` и `SPEC`,
- извлекает онлайн, название карты и ссылку на изображение карты,
- публикует в Discord одно служебное сообщение и затем **редактирует его**, не создавая новые сообщения,
- хранит `message_id` и последнее успешное состояние в JSON-файле.

## Стек
- Python 3.11+
- requests
- beautifulsoup4
- discord.py
- asyncio
- logging

## Структура
- `main.py` — запуск бота
- `bot/config.py` — конфигурация из env
- `bot/parser.py` — парсер HTML-структуры карточек
- `bot/embeds.py` — формирование Discord embeds
- `bot/widget_updater.py` — цикл обновлений и работа с сообщением
- `bot/state.py` — JSON-хранилище состояния

## Настройка
1. Создайте и заполните `.env` по примеру:
   ```bash
   cp .env.example .env
   ```
2. Установите зависимости:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
3. Запуск:
   ```bash
   python main.py
   ```

## Переменные окружения
- `DISCORD_TOKEN` — токен бота (обязательно)
- `CHANNEL_ID` — ID канала для виджета (обязательно)
- `UPDATE_INTERVAL_SECONDS` — интервал обновления, по умолчанию `180`
- `BASE_URL` — URL источника, по умолчанию `https://breaking.proxy.sqstat.ru`
- `STATE_FILE` — путь к JSON-состоянию, по умолчанию `state.json`
- `LOG_LEVEL` — уровень логов (`INFO` по умолчанию)

## Поведение при сбоях
- Если сайт временно недоступен/вернул пустой ответ — ошибка пишется в лог, бот продолжает цикл.
- Если нужная карточка не найдена — возвращается пустая структура для этой карточки, ошибка логируется.
- Если карта/изображение отсутствуют — поля карты остаются пустыми без падения.
- Если сообщение по сохранённому `message_id` не найдено — бот создаёт новое и сохраняет новый ID.
- Последнее успешное состояние сохраняется локально и поднимается после рестарта.

## Деплой на bothost.ru
1. Создайте приложение/контейнер с Python 3.11+.
2. Загрузите проект, добавьте `.env` с переменными.
3. Установите зависимости: `pip install -r requirements.txt`.
4. Команда запуска: `python main.py`.
5. Убедитесь, что `STATE_FILE` указывает на доступный путь записи (например, `state.json`).

Проект не требует БД/Redis и работает одним процессом.
